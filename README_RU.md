# Heat Island Optimizer Android

Офлайн-версия оптимизатора островов для Android.

Главное отличие от Flask-версии: Python-сервера больше нет. Расчёт, авторазмещение, каталог построек, шаблоны островов и сохранения работают прямо внутри приложения через HTML/CSS/JS.

## Что внутри

```text
heat_island_optimizer_android/
├─ capacitor.config.json
├─ package.json
├─ 01_install_deps.bat
├─ 02_add_or_sync_android.bat
├─ 03_open_android_studio.bat
├─ 04_build_debug_apk.bat
├─ README_RU.md
└─ www/
   ├─ index.html
   ├─ style.css
   ├─ catalog.js
   ├─ templates.js
   ├─ optimizer.js
   └─ app.js
```

## Что реализовано

- Полностью офлайн.
- Android APK через Capacitor.
- Ручное рисование острова.
- Клетки вне острова — синие.
- Кусты/камни — часть острова, но временно заблокированы.
- Радиаторы — белые.
- Источники, генераторы, радиаторы выбираются из игровых списков.
- Ручного ввода мощности нет.
- Поиск по типам построек через красивые выпадающие поля.
- Уровни запоминаются отдельно для каждого типа постройки.
- Ветротурбина работает без генератора.
- Ветротурбина требует охлаждения 20% от своей выработки.
- Обычные источники идут через генераторы.
- Обычные генераторы создают 25% отработанного тепла.
- Источник делит тепло поровну между всеми соседними генераторами.
- Радиатор может охлаждать несколько генераторов/ветротурбин.
- Авторазмещение ограничено 5000 попытками.
- Шаблоны островов встроены в приложение и не удаляются пользователем.
- Пользовательские сохранения хранятся локально на устройстве.
- Есть экспорт/импорт JSON.

## Установка инструментов на ПК

Нужно установить:

1. Node.js LTS.
2. Android Studio.
3. Android SDK внутри Android Studio.

После установки Android Studio желательно один раз открыть его и дождаться, пока он поставит SDK/Gradle/прочие компоненты.

## Первая сборка APK

Открой терминал или cmd в папке проекта.

### 1. Установить зависимости

```bat
01_install_deps.bat
```

Или вручную:

```bat
npm install
```

### 2. Создать Android-проект

```bat
02_add_or_sync_android.bat
```

Или вручную:

```bat
npx cap add android
```

Если папка `android/` уже есть, команда из bat-файла сделает `npx cap sync android`.

### 3. Открыть Android Studio

```bat
03_open_android_studio.bat
```

Или вручную:

```bat
npx cap open android
```

### 4. Собрать debug APK

Вариант через bat:

```bat
04_build_debug_apk.bat
```

Итоговый файл будет примерно здесь:

```text
android/app/build/outputs/apk/debug/app-debug.apk
```

Его можно скинуть на Android-телефон и установить. Телефон может попросить разрешить установку из неизвестных источников.

## Сборка release APK через Android Studio

В Android Studio:

```text
Build → Generate Signed Bundle / APK
→ APK
→ Create new keystore
→ заполнить пароль/alias
→ release
→ Finish
```

Итоговый release APK будет в папке вывода Android Studio.

Keystore не теряй. Если потом захочешь обновлять приложение с тем же package id, новая версия должна быть подписана тем же ключом.

## Как добавить чистовой остров в APK

1. Запусти текущую версию приложения/страницы.
2. Нарисуй чистовой остров.
3. Разметь кусты/камни.
4. Постройки лучше очистить.
5. Нажми `Показать JSON`.
6. Скопируй объект JSON.
7. Открой файл:

```text
www/templates.js
```

8. Вставь объект в массив `window.HIO_TEMPLATES`.

Пример структуры:

```js
window.HIO_TEMPLATES = [
    {
        id: "island_01",
        name: "Остров 01",
        rows: 20,
        cols: 20,
        buildable: [...],
        obstacles: [...],
        buildings: [...],
        allow_obstacles: true,
        params: {...}
    }
];
```

После изменения `www/templates.js` нужно выполнить:

```bat
02_add_or_sync_android.bat
```

А потом пересобрать APK.

## Что менять при обновлении веб-части

Если меняешь файлы в `www/`, после этого нужно выполнить:

```bat
npx cap sync android
```

И только потом собирать APK.

## Где хранятся сохранения на телефоне

Пользовательские сохранения лежат локально в WebView/localStorage приложения.

Это значит:

- они доступны без интернета;
- они не уходят на сервер;
- они могут пропасть при очистке данных приложения;
- при удалении приложения они тоже обычно удаляются;
- для переноса есть экспорт/импорт JSON.

## Если APK не собирается

Частые причины:

1. Не установлен Node.js.
2. Не установлен Android Studio/SDK.
3. Не выполнен `npm install`.
4. Не выполнен `npx cap add android`.
5. После изменения `www/` забыли выполнить `npx cap sync android`.
6. Gradle при первой сборке долго скачивает зависимости — это нормально.
